# GitHub Profile Asset Requirements

## Purpose

This document defines the image assets needed for the `SuperShawnSi` GitHub profile refresh.
The current profile README has been simplified to a text-first version. The next upgrade step is to replace parts of it with stronger visual assets prepared manually.

The target tone is:

- calm
- restrained
- editorial
- photography-led
- technically polished

Avoid flashy cyberpunk, gaming-poster aesthetics, cluttered collages, or heavy neon color usage.

## Delivery Folder

Place finished files under:

```text
SuperShawnSi/assets/
```

Use these exact filenames so they can be wired into the README without additional renaming.

## Required Assets

### 1. `profile-banner.png`

**Purpose:**
Primary hero image for the top of the GitHub profile README.

**Recommended size:**

```text
1600 x 520 px
```

**Format:**

- PNG preferred
- JPG acceptable if export quality is high

**Content guidance:**

- One strong photography-led hero composition
- Can include one of these directions:
  - a single best photograph with generous negative space
  - a portrait + camera + one photograph in a restrained collage
  - a minimal composition mixing personal identity and photographic work
- Optional text can be embedded into the image, but keep it minimal

**Should communicate:**

- this is a personal brand page
- the owner is both a builder and a photographer
- the tone is mature and intentional

**Visual rules:**

- use a soft, clean background or controlled photographic contrast
- keep typography sparse if included
- prefer neutral colors: off-white, gray, charcoal, muted blue, warm beige
- do not overdecorate

### 2. `ppass-cover.png`

**Purpose:**
Featured project image for `PP@SS`.

**Recommended size:**

```text
1400 x 900 px
```

**Format:**

- PNG preferred

**Content guidance:**

- Show `PP@SS` as a real product, not just code
- Best options:
  - one clean homepage screenshot
  - a 2 to 3 frame product collage
  - homepage + detail page + about page composition

**Must highlight:**

- interface quality
- photography presentation
- product maturity

**Avoid:**

- terminal screenshots
- raw code screenshots
- too many tiny thumbnails
- unreadable mockup-heavy compositions

### 3. `photo-strip.png`

**Purpose:**
Optional photography strip for a later README enhancement.

**Recommended size:**

```text
1500 x 420 px
```

**Format:**

- PNG preferred

**Content guidance:**

- 3 to 4 photographs arranged horizontally
- consistent color treatment
- should feel like a curated contact sheet or portfolio strip

**Use case:**

- reinforce the photography side of the profile without making the page too busy

## Production Guidance

If creating these in Figma:

- use one frame per asset
- export at 1x or 2x depending on clarity
- keep margins intentional
- do not rely on tiny text that becomes unreadable in GitHub rendering

If using screenshots from `PP@SS`:

- prefer the existing homepage, about page, and detail page visuals
- crop for clarity rather than including everything
- preserve generous whitespace where possible

## Acceptance Criteria

Assets are ready when:

- filenames match exactly
- dimensions are close to the recommended sizes
- images still look clear when scaled down in a GitHub README
- the visual direction matches the profile tone
- the banner and project cover feel like parts of the same personal brand

## Handoff

After the files are prepared, notify Codex with either:

- the asset filenames if they are placed in `SuperShawnSi/assets/`
- or the exact file paths if they live elsewhere temporarily

Once provided, the next step is to replace the text-only README sections with image-backed sections.
